Unigram uses **NuGet** for managed (.NET) dependencies and **vcpkg** for native (C++) libraries. Some components ÔÇö TDLib, LibVLC, and WebRTC ÔÇö are built separately and distributed as NuGet packages placed in the `Libraries` folder.

## Native dependencies (vcpkg)

These libraries are built via vcpkg and targeted to UWP (`x64-uwp` and `arm64-uwp`). See [Build instructions](/contributing/build-instructions) for the full installation steps.

Install all native libraries for both architectures:

```powershell
./vcpkg.exe install ffmpeg[dav1d,opus,vpx]:x64-uwp lz4:x64-uwp openssl:x64-uwp zlib:x64-uwp libogg:x64-uwp opus:x64-uwp boost-regex:x64-uwp
./vcpkg.exe install ffmpeg[dav1d,opus,vpx]:arm64-uwp lz4:arm64-uwp openssl:arm64-uwp zlib:arm64-uwp libogg:arm64-uwp opus:arm64-uwp boost-regex:arm64-uwp
./vcpkg.exe integrate install
```

<Note>
  You can install only the architecture you need (x64 or arm64) rather than both.
</Note>

## LibVLC

Unigram uses LibVLC for video and audio playback. The system-provided media player does not meet the app's quality requirements, so LibVLC is bundled directly.

- **Version:** `3.0.22-rc1` with additional patches applied on top
- **Build script:** `Unigram repository\Libraries\vlc\build.ps1`
- **Requires:** [Docker Desktop for Windows](https://docs.docker.com/desktop/setup/install/windows-install/) installed and running

To build LibVLC:

```powershell
powershell -ExecutionPolicy ByPass ./build.ps1 -arch x64,ARM64
```

The script applies the required patches to the LibVLC source (included as a submodule) and produces a NuGet package in the `Libraries` folder. The resulting package corresponds to the `VideoLAN.LibVLC.UWP` NuGet reference.

<Tip>
  If you don't need to modify LibVLC, you can skip this step ÔÇö the pre-built NuGet package is already referenced from `nuget.org`.
</Tip>

## WebRTC

Unigram uses WebRTC for voice calls and video chats. Because WebRTC does not officially support UWP, Unigram uses a custom UWP fork.

- **Source:** `Unigram repository\Libraries\webrtc`
- **Acquire script:** `acquire.cmd` ÔÇö clones the WebRTC source to `C:\webrtc` (~1.5 hours)
- **Build script:** `build.cmd "$arch$" "$config$"`

<Warning>
  Building WebRTC is resource-intensive and can take over 90 minutes. For most contributors, building [without WebRTC](/contributing/build-instructions#building-without-webrtc-skip-calls-support) is recommended unless you are specifically working on calls or video chat features.
</Warning>

Build steps:

1. Open **Start Menu Ôćĺ Visual Studio 2022 Ôćĺ x64 Native Tools Command Prompt for VS 2022**.
2. Navigate to `.\Unigram\Libraries\webrtc`.
3. Run `.\acquire.cmd` to clone the WebRTC source code.
4. Run `.\build.cmd "$arch$" "$config$"`, replacing `$arch$` with `x64`, `win32`, or `arm64` and `$config$` with `release` or `debug`.

<Note>
  The WebRTC build instructions are based on [WinRTC](https://github.com/microsoft/winrtc/tree/master/patches_for_WebRTC_org/m84).
</Note>

## TDLib

Unigram communicates with Telegram servers through TDLib, the official Telegram client library.

- **Source:** Built separately from the [tdlib/td](https://github.com/tdlib/td) repository
- **Distribution:** A `.nupkg` file placed in `Unigram\Libraries`, resolved by NuGet via the local `Libraries` package source

See [TDLib setup](/contributing/tdlib-setup) for the full build guide.

## Architecture support

Unigram supports the following build targets:

| Architecture | vcpkg triplet | Use case |
|---|---|---|
| x64 | `x64-uwp` | 64-bit Intel/AMD desktops and laptops |
| ARM64 | `arm64-uwp` | ARM-based devices (e.g., Snapdragon X) |
